Modules in Signalize doesn't have to be only some set of functions. It can be even an entire page like `homepage` or `articles` or some sections like galeries and such.

No matter what you plan to create, the definition is always the same.

## Definition
Empty plugin function looks like this:
```js
// Plugin can be without config
export default async (signalizeInstance, pluginConfig) => {
	const myMethod = () =>  alert('Hello World!');

	// You can return several functions
	// or nothing at all
	return { myMethod };
}
```

## Usage

### Instant Initialization
The instant initialization is done directly during creating a new Signalize instance:
```js
import Signalize from 'signalizejs';
import myModule from 'path/to/my-module.js';

const { myMethod } = new Signalize({
	modules: [
		['my-module', myModule, {/* Direct Config - optional */}]
	],
	params: {
		'my-module': { /* Params Config - optional */}
	}
});
```

### Lazy Initialization
When defining a new module, you need to first configure the import map and then import the module:
```html
<script type="importmap">
	{
		"imports": {
			"my-module": "/path/to/my-module.js"
		}
	}
</script>
<script type="module">
	import Signalize from 'signalizejs';

	const signalize = new Signalize({
		params: {
			'my-module': {/* Params Config - optional */}
		}
	});

	const { myMethod } = await signalize.resolve('my-module');

	// Or with direct config for a new separated instance
	const { myMethod } = await signalize.resolve([
		'my-module', { /* Direct Config */}
	]);
</script>
